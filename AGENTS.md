# Global Agent Rules

Single source of truth for agent behavior in this repo.

## Code

- Write the smallest change that solves the problem.
- Reuse existing code instead of duplicating logic.
- Avoid speculative abstraction and extra layers.
- Match the surrounding conventions and naming.

## Human-in-the-Loop

- Check the standing HITL rulings before asking a question.
- For difficult design work, prefer Claude Fable 5 first when available.

## Port Management

- Read `PORTMASTER.md` before starting or reconfiguring a service.
- Record new active services there as soon as a port changes.

## State Management

- Keep `HANDOFF.md` current while work is in progress.
- Use `SUMMARY.md` as the repo source map before broad exploration.
