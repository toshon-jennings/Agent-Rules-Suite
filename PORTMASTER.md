# PORTMASTER

Port guidance for services and local development.

## Rules

- Check this file before assigning a port.
- Avoid `0.0.0.0` unless external access is required.
- Keep high-use services in stable, documented ranges.
- Update this file whenever a service port changes.

## Notes

- Prefer ports that do not collide with existing local apps.
- Keep service ownership explicit so conflicts are easy to resolve.
